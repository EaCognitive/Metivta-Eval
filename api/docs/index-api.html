📝 How It Works

When you submit your endpoint URL to /submit, we will:

  1. Call your endpoint 100 times (once for each evaluation question)
  2. Send a POST request with a question
  3. Expect a JSON response with an answer
  4. Evaluate your answers using all evaluators
  5. Return your scores and update the leaderboard

✅ Your Endpoint Must:

Accept POST requests with this format:

{
  "question": "What is the source for lighting Shabbat candles?"
}

And return responses in this format:

{
  "answer": "The obligation to light Shabbat candles is derived from the Talmud..."
}

POST /submit Submit Evaluation Run AUTH ›

Submit your Torah Q&A endpoint for comprehensive evaluation of ALL 100 questions

How Submission Works

The evaluation process is fully automated. You submit your endpoint URL once, and we handle the rest.

Step 1: You submit your endpoint URL via POST request

Step 2: Our system automatically queries your endpoint 100 times

Step 3: Each query contains a single question in JSON format

Step 4: Your endpoint returns an answer for each question

Step 5: All responses are evaluated using our evaluators

Step 6: Aggregate scores are calculated and returned immediately

The entire evaluation runs synchronously. Your submission request will complete once all 100 questions have been processed.

What YOU Submit (One-time submission)

POST https://metivta-eval.onrender.com/submit
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{
  "author": "Your Name",
  "system_name": "Your System Name",
  "endpoint_url": "https://your-api.com/answer"
}

Request Parameters

author REQUIRED string
Your name or team name
system_name REQUIRED string
Name of your system
endpoint_url REQUIRED string
Your API endpoint URL that will answer all 100 questions
