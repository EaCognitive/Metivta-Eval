# Contributing to the Open Torah Standard

MetivtaEval is built around a community-governed standard for AI-driven Torah scholarship. We welcome contributions to our public dataset and rubrics.

## Proposing Changes

All changes are managed through GitHub Issues and Pull Requests.

1. **Proposal:** To suggest a new question, a better ground-truth answer, or a change to a rubric, please open a new **GitHub Issue**.
2. **Community Polling:** We use emoji reactions to gauge community support. If you agree with a proposal, please add a 👍 reaction to the original issue post.
3. **Review Threshold:** Any proposal that receives **50 or more 👍 reactions** within 21 days will be formally reviewed by the project maintainers.
4. **Decision:** After review, a maintainer will either create a Pull Request to implement the change or post a comment explaining why it was not accepted.
